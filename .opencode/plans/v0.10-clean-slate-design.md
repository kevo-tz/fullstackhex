## Approaches Considered

### Approach A: Sequential clean — one item per commit (CHOSEN)
A10 → S7 → S8 → S11. Each item lands as an independent PR-ready commit. Order: auth UI first (visible), then S7 (small), then S8 (heavy), then S11 (capstone). Clean history, easy review.

### Approach B: Parallel — subagent for S8
Spawn subagent for coverage while main thread handles A10+S11. S7 last. Faster wall clock but harder to coordinate. Risk: subagent test patterns diverge from inline style.

### Approach C: Two releases
A10+S7 ships as v0.10.0. S8+S11 ships as v0.10.1. Safer but delays clean slate.

## Recommended Approach

Sequential. Each commit is self-contained. S7 is manual multipart (already implemented — commit is just integration tests + documentation). S8 uses `cargo tarpaulin --ignore-tests` for coverage measurement.

### Rollback criteria
If any commit breaks `make test-e2e` or `cargo test`, revert it before proceeding to the next commit. Sequential execution means each commit lands on a known-good base.

### Detailed execution plan

**Commit 1 — A10: Auth UI finalize**

- Auth-gated page at `/dashboard`: shows user email, logout button, placeholder for future features. Redirect to `/login` if unauthenticated. Uses existing `localStorage` JWT check from `Layout.astro`.
- Login redirects to `/dashboard` on success (not `/` — avoids flash of unauthenticated state).
- Logout: clear `localStorage` tokens, redirect to `/login`.
- Client-side token refresh: Astro `<script>` island intercepts `fetch` 401 responses. Calls `/api/auth/refresh`, stores new access token in `localStorage`, retries original request. On refresh failure (expired refresh token, revoked session): clear auth state, redirect to `/login`.
- Token storage: access token in `localStorage` (matching existing `AuthForm.astro` convention). Refresh token in `localStorage` — acceptable for template project; httpOnly cookie would require backend changes out of scope for this commit.
- Vitest component tests: `frontend/tests/vitest/auth-form.vitest.ts` (new). Tests: form submit with valid/invalid credentials, validation error display, OAuth provider button rendering (conditional on env).
- Files:
  - `frontend/src/pages/dashboard.astro` (new) — auth-gated page
  - `frontend/src/pages/login.astro` (modify) — redirect to `/dashboard` after login
  - `frontend/src/components/AuthForm.astro` (modify) — add token refresh interceptor
